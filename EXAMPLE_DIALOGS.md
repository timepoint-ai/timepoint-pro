## Reading Guide

### Per-Turn Generation

This transcript was generated by the **per-turn dialog engine**, where each character's turn is produced by an independent LLM call with persona-derived generation parameters. This is a structural improvement over the legacy single-call approach (used in earlier runs), where one model wrote all four characters in a single completion. The per-turn architecture means:

- **Temperature** is derived from each character's arousal state (higher arousal produces more varied language)
- **Max tokens** is derived from energy level (higher energy produces longer turns)
- **Frequency and presence penalties** are derived from the character's behavior vector (preventing cross-character phrase bleeding)
- A **LangGraph steering agent** selects speakers and manages narrative arc across the conversation

### How Dialogs Were Generated

Each dialog is produced by:
1. Building **Fourth Wall context** per character: back layer (true emotional state, withheld knowledge, suppressed impulses) and front layer (causally-filtered knowledge, natural-language relationships)
2. Computing **PersonaParams** per character from entity tensor state (arousal &rarr; temperature, energy &rarr; max_tokens, behavior_vector &rarr; frequency/presence penalty)
3. Running a **LangGraph steering loop**: steering agent selects speaker &rarr; character agent generates one turn with persona params &rarr; quality gate evaluates &rarr; loop or end
4. **Post-dialog prospection**: each participant generates personality-filtered episodic memories, updated rumination topics, and knowledge from expectations
5. Recording information transfers as exposure events in the knowledge graph (M19)
6. Syncing emotional state changes back to entity tensors
